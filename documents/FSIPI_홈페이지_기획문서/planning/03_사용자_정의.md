# 사용자 정의

## 1. 사용자 정의 목적

홈페이지는 기관이 보여주고 싶은 내용을 나열하는 방식보다 방문자가 해결하려는 문제를 기준으로 설계해야 한다. 미래스마트산업진흥원의 핵심 사용자는 공공기관, 기업, 교육 수요자, 전문가, 대학·연구기관, 일반 방문자로 구분한다.

## 2. 주요 사용자 그룹

| 우선순위 | 사용자 | 주요 목적 | 핵심 화면 |
|---:|---|---|---|
| 1 | 공공기관·지자체 담당자 | 교육·연구·지역혁신 사업 수행기관 검토, 협력 제안 | 주요사업, 수행사례, 협력문의 |
| 1 | 기업·스타트업 담당자 | 디지털 전환, 교육, 컨설팅, 공동사업, 지원정보 탐색 | 기업지원, 정책정보, 사업제안 |
| 1 | 교육 수요자 | 교육과정·행사 확인과 신청 | 교육과정, 일정, 신청 |
| 2 | 강사·전문가 | 강의, 자문, 프로젝트 참여 기회 확인 | 전문가 등록, 사업정보 |

| 운영 | 관리자·콘텐츠 담당자 | 콘텐츠 등록, 신청·문의 처리, 사이트 운영 | 관리자 대시보드 |

## 3. 핵심 페르소나

### 3.1 지자체 디지털사업 담당자

- 역할: 지역산업, 일자리, 평생교육, 스마트도시 또는 디지털 전환 담당
- 상황: 지역 수요에 맞는 교육·컨설팅·실증 사업 수행기관을 찾고 있음
- 필요한 정보: 기관 실적, 수행 분야, 제안 가능한 사업 모델, 담당자 연락처
- 불편 요소: 추상적인 소개, 실적 근거 부족, 문의 경로 불명확
- 기대 행동: 사업 소개서 다운로드 → 수행사례 확인 → 협력 문의 등록

### 3.2 중소기업 대표 또는 실무책임자

- 역할: 기술개발, 인재 양성, AI 도입, 정부지원사업 대응 담당
- 상황: 내부 인력과 예산이 부족해 외부 전문가와 지원정보가 필요함
- 필요한 정보: 기업 지원 내용, 교육과정, 정부사업공고, 컨설팅 절차
- 불편 요소: 공고 원문이 어렵고 자사에 해당하는지 판단하기 어려움
- 기대 행동: 지원정보 확인 → 관련 사업 문의 → 상담 요청

### 3.3 교육과정 참여 희망자

- 역할: 대학생, 취업준비생, 재직자, 일반 성인 학습자
- 상황: AI·소프트웨어·디지털 업무혁신 교육을 찾고 있음
- 필요한 정보: 대상, 난이도, 일정, 장소, 비용, 선수지식, 결과물, 신청 상태
- 불편 요소: 교육 제목만 있고 실제 학습 내용과 준비물이 불명확
- 기대 행동: 과정 상세 확인 → 신청 → 접수 확인

### 3.4 대학·연구기관 협력 담당자

- 역할: 산학협력단, 연구소, 교수, 연구자
- 상황: 공동 연구, 세미나, 교육과정 개발, 현장 프로젝트 파트너가 필요함
- 필요한 정보: 연구 분야, 전문가 네트워크, 협력 방식, 기존 협약과 사례
- 불편 요소: 기관의 역할과 책임 범위가 모호함
- 기대 행동: 협력 분야 확인 → 제안서 첨부 → 담당자 후속 연락

### 3.5 강사·자문위원·산업전문가

- 역할: AI, 소프트웨어, 스마트산업, 경영, 정책, 교육 전문가
- 상황: 강의, 심사, 자문, 프로젝트 참여 기회를 찾고 있음
- 필요한 정보: 모집 분야, 요구 경력, 개인정보 활용 범위, 참여 방식
- 불편 요소: 등록 후 활용 여부와 연락 기준이 불명확
- 기대 행동: 전문가 등록 → 이력 제출 → 검토 결과 안내

## 4. 관리자 사용자

### 4.1 최고관리자

- 관리자 계정과 권한 관리
- 전체 콘텐츠 관리
- 개인정보와 보안 설정
- 메뉴·배너·기관정보 관리
- 삭제 복구와 감사 로그 확인

### 4.2 콘텐츠 관리자

- 공지, 교육, 사업, 자료, 소식 작성
- 파일과 이미지 업로드
- 게시·예약·비공개 상태 관리
- SEO 제목과 설명 입력

### 4.3 신청·문의 담당자

- 교육 신청 조회
- 협력 문의 분류와 상태 변경
- 답변 기록
- CSV 내보내기
- 개인정보 보유기간에 따른 파기 처리

## 6. 사용자 여정

### 6.1 교육 신청 여정

① 검색 또는 SNS 링크로 교육 상세 화면 진입  
② 교육 대상, 일정, 커리큘럼, 장소, 비용 확인  
③ 신청 가능 상태 확인  
④ 신청서 작성과 개인정보 동의  
⑤ 접수 완료 메시지와 이메일 확인  
⑥ 담당자 승인 또는 안내 수신

### 6.2 협력 제안 여정

① 주요사업 또는 수행사례 확인  
② 협력 가능한 분야 선택  
③ 기관·담당자 정보와 제안 내용 입력  
④ 제안서 파일 첨부  
⑤ 접수번호 확인  
⑥ 담당자 검토 및 연락

### 6.3 정책정보 탐색 여정

① 정책·산업정보 목록 진입  
② 분야, 대상, 기관, 기간 필터 적용  
③ 핵심 요약과 마감일 확인  
④ 원문 링크 또는 첨부파일 확인  
⑤ 관련 교육·컨설팅 화면으로 이동

## 7. 접근성 및 사용 환경

- 모바일 방문 비중을 고려해 핵심 행동 버튼을 화면 하단에서도 쉽게 누를 수 있게 한다.
- 공공기관 사용자를 위해 문서 다운로드와 인쇄 화면을 제공한다.
- 고령 사용자와 일반 성인을 고려해 본문 글자 크기와 대비를 충분히 확보한다.
- 표와 일정은 모바일에서 카드형 또는 가로 스크롤로 제공한다.
- 이미지 안에 핵심 문구를 넣지 않고 HTML 텍스트로 제공한다.
- 모든 입력란에 라벨, 오류 메시지, 필수 표시를 제공한다.

## 8. 사용자 요구 우선순위

### P0: 반드시 필요한 요구

- 기관과 사업을 신뢰할 수 있는 정보
- 모바일에서도 읽기 쉬운 화면
- 교육·행사 일정과 신청 방법
- 게시물 검색과 분류
- 명확한 문의 경로
- 개인정보 동의와 접수 확인

### P1: 조기 확장 요구

- 신청 상태 확인
- 전문가 등록
- 뉴스레터 구독
- 관심 분야별 콘텐츠 추천
- 콘텐츠 인쇄·공유

### P2: 장기 확장 요구

- 개인 맞춤형 대시보드
- AI 검색과 질의응답
- 교육 이력과 수료증
- 협력기관 전용 자료실
- 다국어 지원
