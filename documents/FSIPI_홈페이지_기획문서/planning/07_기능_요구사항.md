# 기능 요구사항

## 1. 우선순위 기준

- P0: 1차 공개에 반드시 필요한 기능
- P1: 공개 직후 운영 효율을 위해 필요한 기능
- P2: 데이터와 사용자가 누적된 뒤 도입할 기능

## 6. 검색 기능

### 6.1 검색 대상

- 주요사업
- 수행사례
- 교육·행사
- 공지사항
- 활동소식
- 정책·산업정보
- 자료실

### 6.2 검색 조건

- 제목
- 요약
- 본문
- 태그
- 출처 기관
- 연도
- 콘텐츠 유형

### 6.3 구현 단계

① MVP: MongoDB text index 또는 정규화된 검색 API  
② 확장: 형태소 분석이 필요한 경우 Elasticsearch 또는 Meilisearch 검토  
③ AI 검색: 임베딩과 RAG를 별도 서비스로 도입

## 7. 파일 관리 기능

- 허용 확장자 화이트리스트
- 이미지: JPG, PNG, WebP, SVG 검토
- 문서: PDF, HWPX, DOCX, XLSX, PPTX, ZIP
- 파일별 최대 용량 설정
- 파일명 정규화
- 악성 파일 검사 또는 업로드 제한
- 공개 파일과 비공개 신청 첨부파일의 저장소 분리
- 삭제 시 참조 여부 확인

| 알림 | 수신자 | 우선순위 |

이메일 발송 실패는 재시도하고 관리자 화면에서 실패 내역을 확인할 수 있게 한다.

## 9. 보안 요구사항

- HTTPS 강제
- Firebase ID Token 또는 동등한 인증 토큰을 백엔드에서 검증
- 관리자 역할 검증
- CORS 허용 도메인 제한
- 입력값 서버 측 검증
- NoSQL Injection 방지
- XSS 방지를 위한 HTML 정제
- 파일 MIME 타입과 확장자 검증
- 로그인·문의 API 요청 제한
- 비밀키는 Render와 Vercel 환경변수에 저장
- 운영 DB 접근 IP와 계정 권한 최소화
- 관리자 변경 로그 저장

## 10. 개인정보 요구사항

- 신청과 문의에 필요한 최소 정보만 수집
- 수집 목적, 항목, 보유기간, 거부권 고지
- 필수 동의와 선택 동의 분리
- 관리자 화면에서 개인정보 마스킹
- 보유기간 만료 데이터 삭제 또는 익명화
- CSV 내보내기 권한 제한
- 다운로드 기록 저장
- 개인정보가 포함된 첨부파일의 공개 URL 차단

## 11. 성능 요구사항

- 메인 대표 이미지는 적절한 크기로 변환
- WebP 또는 AVIF 제공
- 화면 밖 이미지 지연 로딩
- API 응답 페이지네이션
- 목록 캐시
- Vercel CDN 활용
- Render 무료·저가 플랜 사용 시 슬립과 초기 응답 지연을 고려
- 대량 파일은 애플리케이션 서버가 아닌 객체 저장소 사용

## 12. 접근성 요구사항

- 모든 이미지의 대체 텍스트
- 명확한 제목 계층
- 키보드만으로 메뉴와 폼 사용 가능
- 포커스 표시
- 색상만으로 상태를 구분하지 않음
- 오류 메시지를 입력란과 연결
- 동영상 자막
- 자동 재생 최소화
- 표에 제목과 헤더 지정

## 13. 로그·백업 요구사항

- API 오류 로그
- 관리자 로그인 로그
- 콘텐츠 생성·수정·삭제 로그
- 신청 상태 변경 로그
- MongoDB Atlas 자동 백업 설정 확인
- 업로드 파일 백업 정책
- 월 1회 복구 테스트

## 14. 향후 AI 기능

AI 기능은 기본 홈페이지가 안정적으로 운영된 후 별도 단계로 추가한다.

- 정책·사업공고 핵심 요약
- 콘텐츠 자동 태깅
- 관련 콘텐츠 추천
- 자연어 통합 검색
- 문의 분류 보조
- 관리자 초안 작성 보조

AI 출력은 자동 게시하지 않고 관리자의 검토와 승인 후 공개한다.
