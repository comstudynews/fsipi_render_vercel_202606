# 벤치마킹 분석

## 1. 분석 대상과 방법

분석 대상은 사용자가 내려받은 `svakorea.org.zip`의 공개 웹페이지다. 압축파일에는 HTML 458개를 포함해 총 817개 파일이 있었으며, 메인 페이지, 기관 소개, 사업, 회원·협력기관, 정책·제도, 정부사업공고, 자료실, 협업 문의를 중심으로 분석했다. 대량 게시글은 전체 내용을 복제하지 않고 목록 구조와 일부 표본만 확인했다.

## 2. 참고 사이트의 정보구조

### 2.1 상위 메뉴

|---|---|
| 협회소개 | 협회개요, 인사말, 조직구성, 오시는길 |
| 주요사업 | 한국형 스마트빌리지 개요, 사업분야 |
| 회원사 | 가입안내, 가입신청, 회원사현황, 회원사혜택, 협력기관 |
| 알림마당 | 협회소식, 회원사소식, 정책·제도, 정부사업공고, 사업·공지, 자료실, 협업제안문의 |

### 2.2 메인 화면의 핵심 구성

- 기관명과 전체 메뉴
- 통합 검색
- 정부사업공고 최신 글
- 정책·제도 최신 글
- 자료실 최신 글

### 2.3 콘텐츠 규모

- 정책·제도: 수백 건
- 정부사업공고: 수백 건
- 자료실: 수백 건
- 협회소식과 사업·공지: 수십 건
- 회원사소식: 이미지·카드형 콘텐츠

이 구조는 콘텐츠가 충분히 누적된 기관에는 효과적이지만, 신설 기관이 처음부터 동일한 게시판 수를 운영하면 빈 화면이 많아지고 관리 부담이 커질 수 있다.

## 3. 강점

### 3.1 기관형 홈페이지의 필수 요소가 갖춰져 있음

기관 소개, 사업, 조직, 오시는길, 정책정보, 자료실, 문의, 약관과 개인정보 처리방침을 한 사이트 안에 배치했다. 미래스마트산업진흥원도 이 기본 골격은 채택할 가치가 있다.

### 3.2 메인 화면에서 최신 정보를 바로 보여줌

정부사업공고, 정책·제도, 자료실을 메인에서 각각 노출해 방문자가 새 정보를 빠르게 확인할 수 있다. FSIPI 홈페이지도 `공지`, `교육·행사`, `정책·산업정보`를 메인에 분리해 보여주는 편이 좋다.

### 3.3 사업분야를 카드형으로 시각화

디지털 생태계, 인재 양성, 협력 네트워크, 데이터, 정책, 교육·행사, 전문가 양성, 글로벌 네트워크 등 사업을 이미지 카드로 제시한다. 사업이 여러 개인 기관에 적합한 방식이다.

### 3.4 협력과 참여 경로가 존재함

가입신청, 협력기관, 협업 문의 등 이해관계자가 다음 행동으로 이동할 수 있다. FSIPI는 이를 교육 신청, 사업 제안, 전문가 등록, 기업 지원 문의로 전환한다.

### 3.5 게시판 검색과 상세 페이지 구조가 일관됨

게시판별 검색, 정렬, 목록, 상세 화면이 반복되는 구조라 사용자가 쉽게 익힐 수 있다. 운영 측면에서도 공통 게시판 컴포넌트로 구현하기 좋다.

## 4. 개선이 필요한 점

### 4.1 협회형 메뉴를 진흥원에 그대로 적용하기 어려움

회원사 모집과 혜택이 큰 비중을 차지한다. 미래스마트산업진흥원은 회원사보다 사업 참여자, 교육생, 기업, 전문가, 대학·연구기관, 공공기관과의 협력이 중심이다. 따라서 `회원사`를 `참여·협력`으로 바꾸는 것이 적절하다.

### 4.2 게시판이 지나치게 세분화됨

협회소식, 회원사소식, 정책·제도, 정부사업공고, 사업·공지, 자료실이 각각 별도 게시판이다. 운영자가 적은 초기에는 콘텐츠가 중복되거나 어느 게시판에 올릴지 모호해질 수 있다.

FSIPI 초기 권장 구조는 다음과 같다.

- 공지사항
- 교육·행사
- 정책·산업정보
- 자료실
- 활동소식

### 4.3 대량 외부자료 수집이 기관 고유 콘텐츠를 가릴 수 있음

정부사업공고와 정책자료가 수백 건 누적되어 있으나, 기관이 직접 수행한 사업과 성과보다 외부 자료가 더 크게 보일 위험이 있다. FSIPI는 외부자료의 양보다 선별 기준, 요약, 활용 포인트를 강조해야 한다.

### 4.4 외부 신청 폼은 사용자 경험이 끊김

가입 신청이 외부 Google Form으로 연결된다. 초기에는 빠르게 시작할 수 있지만 기관 브랜드, 개인정보 고지, 신청 상태 관리가 분리된다. FSIPI MVP에서도 외부 폼을 임시로 사용할 수 있으나, 2차에는 자체 신청 폼과 관리자 조회 기능으로 전환하는 것이 좋다.

### 4.5 정보 중복과 표기 불일치 가능성

압축본에서 페이지별 제목이 반복되고, 본문과 푸터의 연락처가 다르게 나타나는 부분이 있었다. 운영 데이터와 화면 문구를 각각 직접 입력하면 이런 문제가 생긴다. FSIPI는 기관 기본정보를 한 곳에서 관리하고 모든 화면이 같은 데이터를 참조하게 해야 한다.

### 4.6 로그인과 가입 기능의 목적이 명확하지 않음

일반 방문자에게 `INTRANET`, `JOIN`이 노출되지만 어떤 서비스를 제공하는지 즉시 알기 어렵다. FSIPI 초기에는 관리자 로그인만 숨겨진 경로로 제공하고, 일반 회원가입은 실제 필요가 생길 때 도입한다.

### 4.7 접근성과 모바일 사용성의 별도 검증이 필요함

대체 텍스트, 키보드 이동, 명도 대비, 모바일 메뉴, 표의 가로 스크롤을 명시적으로 테스트해야 한다. 디자인 완성도보다 실제 정보 접근성을 우선한다.

## 6. FSIPI 차별화 방향

### 6.1 기관의 고유 전문성을 첫 화면에서 명확히 제시

메인 헤드라인은 추상적인 미래 표현보다 실제 수행 분야를 보여준다.

> AI·디지털 전환 교육, 스마트산업 연구, 기업과 지역의 혁신을 연결합니다.

### 6.2 사업 결과 중심 콘텐츠

- 교육 운영 인원
- 수행 프로젝트
- 협력기관
- 발간 자료
- 기업 지원 건수
- 주요 성과 사례

초기에는 수치가 적더라도 검증 가능한 실적만 표시한다.

### 6.3 교육·연구·기업 지원의 연결

단순 게시판이 아니라 `정책 정보 → 교육과정 → 프로젝트 → 기업 지원 → 성과 사례`가 연결되도록 콘텐츠를 설계한다.

### 6.4 신뢰 가능한 정보 큐레이션

외부 정책자료는 단순 복사보다 다음 항목을 제공한다.

- 핵심 요약
- 대상
- 지원 내용
- 신청 기간
- 원문 출처
- FSIPI 관점의 활용 분야

### 6.5 향후 AI 기능을 고려한 구조화

게시물에 분류, 태그, 기관, 대상, 지역, 기간, 첨부파일, 출처 URL을 구조적으로 저장한다. 이 데이터는 향후 검색, 추천, 챗봇, 자동 요약에 활용할 수 있다.

## 7. 결론

참고 사이트의 장점은 기관형 메뉴, 사업 카드, 최신글 노출, 자료 축적, 협력 문의 구조다. 반면 회원사 중심 구조, 과도한 게시판 수, 외부자료 대량 수집은 미래스마트산업진흥원의 초기 운영에는 맞지 않는다. FSIPI 홈페이지는 `기관 신뢰`, `사업 소개`, `교육·행사`, `정책·자료`, `참여·협력`의 다섯 축으로 단순하게 시작하고, 실제 콘텐츠가 늘어날 때 기능을 확장하는 방식이 적절하다.
